# Roadmap

## Phase 1 - Community Foundation

- Open the community repository
- Define the community scope
- Create the initial glossary
- Define initial research questions
- Start paper review discussions
- Open the first beginner-friendly issues

## Phase 2 - Failure Taxonomy

- Define Physical AI failure modes
- Create an initial taxonomy for drones, robotics, and world models
- Collect examples from the community
- Discuss what makes an AI-generated action physically invalid

## Phase 3 - First Toy Benchmarks

- Create a simple drone route admissibility scenario
- Create a simple robot action admissibility scenario
- Define basic metrics for valid and invalid actions
- Share notebooks and examples

## Phase 4 - Community Reports

- Publish the first community technical note
- Credit contributors
- Summarize open research questions
- Define the next research focus
